# Codebase Structure

**Analysis Date:** 2026-01-22

## Directory Layout

```
living-to-die-site/
├── src/                    # Source code root
│   ├── components/         # Reusable Astro components
│   ├── content/            # Content collections (Markdown)
│   │   ├── blog/           # Blog posts
│   │   ├── poetry/         # Poetry pieces
│   │   ├── publications/   # Published articles
│   │   └── resources/      # External organizations
│   ├── layouts/            # Page layout templates
│   ├── pages/              # Route pages (auto-routed)
│   │   ├── admin/          # Admin interface
│   │   ├── api/            # API endpoints
│   │   ├── blog/           # Blog routes
│   │   └── poetry/         # Poetry routes
```

## Directory Purposes

**src/components/**
- Purpose: Reusable Astro components
- Contains: `Navigation.astro`, `Footer.astro`, `ResourceCard.astro`
- Key files: All components used across multiple pages
- Subdirectories: None (flat structure)

**src/content/**
- Purpose: Content collections with Zod schemas
- Contains: Markdown files with frontmatter metadata
- Key files: `config.ts` (schema definitions)
- Subdirectories:
  - `blog/` - Blog posts with categories and tags
  - `poetry/` - Poetry and reflections
  - `publications/` - Published essays and articles
  - `resources/` - External support organizations

**src/layouts/**
- Purpose: Page wrapper templates
- Contains: `BaseLayout.astro` (main layout)
- Key files: `BaseLayout.astro` - nav, footer, meta tags, Netlify Identity
- Subdirectories: None

**src/pages/**
- Purpose: File-based routing (auto-generates routes)
- Contains: Page components and API endpoints
- Key files: `index.astro` (homepage), `resources.astro`
- Subdirectories:
  - `admin/` - Admin interface (`index.astro`)
  - `api/` - Server endpoints (`og-image.ts`)
  - `blog/` - Blog listing and detail (`index.astro`, `[...slug].astro`)
  - `poetry/` - Poetry listing (`index.astro`)

**src/styles/**
- Purpose: Global CSS and design system
- Contains: CSS files with custom properties
- Key files: `global.css` (current), `global-v2.css` (alternate design)
- Subdirectories: None

**public/**
- Purpose: Static assets served as-is
- Contains: Images, favicon, CMS configuration
- Key files: `book-cover.webp`, `hero-background.jpg`
- Subdirectories:
  - `admin/` - Decap CMS (`index.html`, `config.yml`)

## Key File Locations

**Entry Points:**
- `src/pages/index.astro` - Homepage
- `src/pages/rss.xml.js` - RSS feed generation
- `astro.config.mjs` - Build configuration

**Configuration:**
- `astro.config.mjs` - Astro settings, integrations
- `tsconfig.json` - TypeScript configuration
- `netlify.toml` - Deployment settings
- `public/admin/config.yml` - Decap CMS configuration

**Core Logic:**
- `src/content/config.ts` - Content schema definitions
- `src/pages/api/og-image.ts` - Image proxy endpoint
- `src/layouts/BaseLayout.astro` - Page wrapper

**Testing:**
- Not configured (no test directory)

**Documentation:**
- `README.md` - Project overview (if exists)
- `src/styles/style-guide.md` - Design documentation

## Naming Conventions

**Files:**
- PascalCase for Astro components: `Navigation.astro`, `BaseLayout.astro`
- kebab-case for config files: `og-image.ts`, `global-v2.css`
- Bracket notation for dynamic routes: `[...slug].astro`

**Directories:**
- lowercase for all directories: `components/`, `layouts/`, `pages/`
- Plural for collections: `blog/`, `resources/`

**Special Patterns:**
- `index.astro` - Default page for directory route
- `[...slug].astro` - Catch-all dynamic route
- `*.xml.js` - XML endpoint (RSS, sitemap)

## Where to Add New Code

**New Feature:**
- Primary code: `src/pages/feature-name.astro` or `src/pages/feature-name/index.astro`
- Tests: Not configured
- Config if needed: `astro.config.mjs`

**New Component:**
- Implementation: `src/components/ComponentName.astro`
- Types: Props interface in component file
- Tests: Not configured

**New Content Collection:**
- Schema: Add to `src/content/config.ts`
- Content: Create `src/content/collection-name/` directory
- Page: Create `src/pages/collection-name/index.astro`

**New API Endpoint:**
- Implementation: `src/pages/api/endpoint-name.ts`
- Export: `GET`, `POST`, etc. as named exports

**Utilities:**
- Shared helpers: `src/lib/` (create if needed)
- Type definitions: Inline in components or separate `.d.ts`

## Special Directories

**.astro/**
- Purpose: Astro-generated type definitions
- Source: Auto-generated by Astro build
- Committed: No (gitignored, but types.d.ts may be tracked)

**dist/**
- Purpose: Build output for deployment
- Source: Generated by `npm run build`
- Committed: No (gitignored)

**public/admin/**
- Purpose: Decap CMS interface and configuration
- Source: Static files served at `/admin`
- Committed: Yes
