  - Body: markdown content

**File creation approach:**
Since this is SSR on Vercel, we cannot write to filesystem at runtime. Instead:
- Store newsletter posts in Upstash Redis (similar to homepage content)
- Key pattern: `newsletter:{slug}` with full post data
- On build, blog pages can optionally pull from Redis OR we accept newsletters only go to email (not persisted to blog)

**Simpler approach for MVP:**
- Focus on MailerLite sending only
- Blog post creation can be manual via existing CMS or future enhancement
- This keeps scope manageable

Implement the MailerLite-only approach:
- POST accepts: subject, preheader, htmlBody (already formatted)
- Calls createAndSendNewsletter
- Returns result

Note: Content persistence to blog is a future enhancement. For now, newsletters are sent via MailerLite and recipients can view online via MailerLite's hosted version.
