<task type="auto">
  <name>Task 3: Fix grid layout for consistent 3-column display</name>
  <files>src/pages/resources.astro</files>
  <action>
Fix .resource-grid CSS to maintain 3-column layout:

Current issue: `grid-template-columns: repeat(auto-fit, minmax(350px, 1fr))` causes
fewer than 3 items to expand to fill space.

Solution: Use explicit 3-column grid with responsive breakpoints:
```css
.resource-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 2rem;
}

@media (max-width: 1024px) {
  .resource-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (max-width: 768px) {
  .resource-grid {
    grid-template-columns: 1fr;
  }
}
```

This ensures cards maintain consistent sizing regardless of item count.
  </action>
  <verify>Grid shows 3 columns on desktop even with 1-2 items in a category</verify>
  <done>Resource grid displays 3 columns on desktop, 2 on tablet, 1 on mobile</done>
</task>
