Create src/lib/drafts.ts with CRUD operations for newsletter drafts:

**Draft Interface:**
```typescript
interface NewsletterDraft {
  id: string;
  subject: string;
  preheader: string;
  issueInfo?: string;
  openingParagraph: string;
  section1Title: string;
  section1Content: string;
  section2Title?: string;
  section2Content?: string;
  ctaUrl?: string;
  ctaText?: string;
  closingMessage: string;
  // Scheduling fields
  scheduledFor?: string; // ISO date string, null = draft only
  status: 'draft' | 'scheduled' | 'sent';
  createdAt: string;
  updatedAt: string;
}
```

**Functions:**
1. `saveDraft(draft: Omit<NewsletterDraft, 'id' | 'createdAt' | 'updatedAt' | 'status'>): Promise<NewsletterDraft>` - Creates new draft with generated ID
2. `updateDraft(id: string, updates: Partial<NewsletterDraft>): Promise<NewsletterDraft | null>` - Updates existing draft
3. `getDraft(id: string): Promise<NewsletterDraft | null>` - Get single draft
4. `listDrafts(): Promise<NewsletterDraft[]>` - List all drafts (sorted by updatedAt desc)
5. `deleteDraft(id: string): Promise<boolean>` - Delete draft
6. `getScheduledDrafts(): Promise<NewsletterDraft[]>` - Get drafts where status='scheduled' and scheduledFor is in the past (for cron)

**Key Patterns:**
- Use nanoid for draft ID generation (install if needed)
- Key format: `draft:{id}` for individual drafts
- Use kv.scanIterator to list all drafts
- Return null/false for not-found cases, don't throw
- Include createdAt/updatedAt timestamps

**Error Handling:**
- If KV not configured (missing env vars), throw clear error
- Wrap KV operations in try/catch, log errors
